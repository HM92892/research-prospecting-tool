# Research & Prospecting Tool — Spec 3: Apollo API Integration

This spec adds real prospect finding to the tool. After the tool generates an ICP profile (which it already does), it will now use Apollo's People Search API to find real people matching that ICP and display them in the ICP & Targeting tab.

---

## What This Changes

Currently, the ICP & Targeting tab shows:
- Target job titles (as tags)
- Target industries (as tags)
- Company size range
- Key buying signals
- Apollo search query (text to copy-paste)
- LinkedIn Sales Nav search query (text to copy-paste)
- ICP reasoning

**After this spec**, the ICP & Targeting tab will ALSO show:
- A list of 10-25 real prospects pulled from Apollo's database, with: name, job title, company name, company size, location, and LinkedIn profile URL
- A "Found X prospects matching your ICP" summary line
- Each prospect displayed in a clean row/card format

## Backend Changes (analyzer.py)

### Add Apollo API integration

Add a new function `search_apollo_prospects()` that takes the ICP profile data and queries Apollo's People Search API.

**Apollo API Details:**
- **Endpoint:** `POST https://api.apollo.io/api/v1/mixed_people/api_search`
- **Auth:** API key passed as query parameter `api_key` OR header `X-Api-Key`
- **Credit cost:** Zero — this endpoint does not consume credits
- **Response:** Returns people records with name, title, company, location, LinkedIn URL

**How to build the API request from the ICP data:**

The `generate_icp_profile()` function already returns structured ICP data including target titles, industries, and company size. Use those fields to build the Apollo query:

```python
import requests

def search_apollo_prospects(icp_profile, api_key):
    """
    Takes the ICP profile generated by Claude and queries Apollo 
    to find real matching prospects.
    """
    
    # Parse the ICP data to extract search parameters
    # The ICP profile contains: target_titles, target_industries, 
    # company_size, buying_signals, apollo_query, linkedin_query, reasoning
    
    url = "https://api.apollo.io/api/v1/mixed_people/api_search"
    
    headers = {
        "Content-Type": "application/json",
        "X-Api-Key": api_key
    }
    
    # Build the request body from ICP data
    body = {
        "page": 1,
        "per_page": 25,
        "person_titles": icp_profile.get("target_titles", []),
        "person_seniorities": ["vp", "director", "head", "manager"],
        "organization_num_employees_ranges": [
            parse_company_size(icp_profile.get("company_size", "50-500"))
        ],
        "q_keywords": " ".join(icp_profile.get("target_industries", []))
    }
    
    try:
        response = requests.post(url, json=body, headers=headers)
        response.raise_for_status()
        data = response.json()
        
        # Extract the people from the response
        prospects = []
        for person in data.get("people", []):
            prospect = {
                "name": person.get("name", "Unknown"),
                "title": person.get("title", "Unknown"),
                "company": person.get("organization", {}).get("name", "Unknown"),
                "company_size": person.get("organization", {}).get("estimated_num_employees", "N/A"),
                "location": person.get("city", "") + ", " + person.get("state", ""),
                "linkedin_url": person.get("linkedin_url", ""),
                "company_website": person.get("organization", {}).get("website_url", "")
            }
            prospects.append(prospect)
        
        return {
            "total_found": data.get("pagination", {}).get("total_entries", 0),
            "prospects": prospects,
            "query_used": body
        }
        
    except requests.exceptions.RequestException as e:
        return {
            "total_found": 0,
            "prospects": [],
            "error": str(e),
            "query_used": body
        }


def parse_company_size(size_string):
    """
    Converts ICP company size like '100-2000 employees' 
    into Apollo format '100,2000'
    """
    import re
    numbers = re.findall(r'\d+', size_string)
    if len(numbers) >= 2:
        return f"{numbers[0]},{numbers[1]}"
    elif len(numbers) == 1:
        return f"{numbers[0]},10000"
    return "50,1000"  # default fallback
```

### Handle the API key

**Option A (recommended for now):** Store the Apollo API key as an environment variable in Replit:
- In Replit, go to "Secrets" (lock icon in the sidebar)
- Add a secret: key = `APOLLO_API_KEY`, value = your actual Apollo API key
- In the code: `api_key = os.environ.get("APOLLO_API_KEY")`

**Option B (for sharing/demo):** Add an optional "Apollo API Key" input field in the UI settings/config area. If no key is provided, the prospect search step is skipped and the ICP tab just shows the search queries as it does now (graceful fallback).

**Implement both:** Use the environment variable by default, but also check for a key passed from the frontend. If neither exists, skip the Apollo search gracefully.

### Graceful fallback

If the Apollo API key is missing or the API call fails:
- Do NOT break the tool or show an error
- Just skip the prospect list and show the ICP tab as it currently works (search queries, titles, industries, etc.)
- Add a subtle note at the bottom: "Connect your Apollo API key to find real prospects matching this ICP"

## Environment Setup

To get this working, you need an Apollo API key:

1. Go to https://app.apollo.io (sign up if you don't have an account — it's free)
2. Go to Settings → Integrations → API → Generate API Key
3. Copy the key
4. In Replit: click the lock icon (Secrets) → add key `APOLLO_API_KEY` with your API key as the value

**Important:** The People Search API endpoint does NOT consume credits. You can make searches for free. It only costs credits when you try to get email addresses or phone numbers (enrichment), which we are NOT doing.
